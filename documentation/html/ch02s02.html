2.2. Getting Pines

There are several sites which host Pines, and Pines related software.

  • https://github.com/sciactive

    The SciActive GitHub page. Pines' repositories are listed here. Each repository here has a bugtracker and a wiki.

  • http://sourceforge.net/projects/pines/

    The Pines project page on SourceForge. This is where Pines' releases are available for download.

  • https://pinesframework.org/ and http://pinescms.org/

    Pines' main websites. The Pines Framework website provides information about Pines Framework, and the Pines CMS website provides information about the CMS build using Pines Framework.

  • http://pinesframework.org/api/

    Pines API Documentation. This can be very helpful to have handy while you are learning Pines development. Before using Pines' advanced features, I recommend reading through the related API documentation to ensure you have a firm understanding of how Pines' features work and how to use them.

  • http://www.sciactive.com/

    Pines' main developer's website.

  • http://www.pinesplaza.com/

    The official Pines Repository. Pines components and templates are hosted and distributed here. You can get a free account and distribute your software through Pines Plaza.

Pines, and its related projects are available in Git repositories from GitHub.

  • https://github.com/sciactive/pines-core

    Pines Core repository. Contains the core system files.

  • https://github.com/sciactive/pines-components

    Pines Components repository. Contains official, SciActive maintained components and templates.

  • https://github.com/sciactive/pform

    Pines Form, a CSS form layout library for HTML and XHTML.

  • https://github.com/sciactive/pgrid

    Pines Grid, a JavaScript (jQuery) data grid plugin.

  • https://github.com/sciactive/pnotify

    Pines Notify, JavaScript notifications for Bootstrap or jQuery UI.

  • https://github.com/sciactive/ptags

    Pines Tags, a JavaScript (jQuery) content tag editor plugin.

  • https://github.com/sciactive/pines-tools

    Build tools, documentation, logo sources, and other various tools.

To get a local copy of Pines for testing or development, we're going to set up a directory of symlinks to the actual repository directories. First, from your server root (or wherever you'd like these directories), clone the Git repositories.

git clone git://github.com/sciactive/pines-core.git
git clone git://github.com/sciactive/pines-components.git

Then create a pines directory, and create links to the core files, components, and templates under the correct directories.

mkdir pines
cd pines

ln -s ../pines-core/* ./
unlink components
unlink templates
unlink media

mkdir components
cd components
ln -s ../../pines-components/com_* ./
cd ..

mkdir templates
cd templates
ln -s ../../pines-components/tpl_* ./
cd ..

mkdir media
cd media
ln -s ../../pines-core/media/* ./
ln -s ../../pines-components/meta_packages/* ./
cd ..

You will need to unlink conflicting components, such as extra entity managers and editors. If you plan to use MySQL and TinyMCE for example, you will need to delete the links to com_pgentity, com_pgsql, and com_ckeditor.

unlink components/com_pgentity
unlink components/com_pgsql
unlink components/com_ckeditor

Now you should be able to access your installation from your server. If you placed it in the server root of your localhost, the address of the development setup would be http://localhost/pines/. Access the site and set up the database following the instructions on screen.

When you set up your database, you should use the username and database name "pines" and the password "password". These are the configuration defaults, so this setup will allow you to use Pines without additional configuration files. Once you set up your database, remove the newly created configuration file in the SQL component you are using (com_mysql or com_pgsql). It is not needed, as long as the configuration is the same as the defaults.

rm pines-components/com_mysql/config.php

You are now ready to start developing and testing. When you first create a user in the new setup, it will be granted administrator rights.

To update your local repository and get the latest code, pull changes from the origin repository (the default remote).

cd pines-core/
git pull

cd ../pines-components/
git pull

I recommend using a Git GUI to make using the repository easier. I use SmartGit (http://www.syntevo.com/smartgit/index.html).
